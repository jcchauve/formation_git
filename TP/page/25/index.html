Itération 25 : Envoyer des modifications

Modifier le dépôt local

Pour partager des modifications avec le dépôt distant, il faut lui envoyer ou "pousser" les commits des modifications à partager.

Nous allons d'abord créer un nouveau commit dans notre branche locale. Modifier le fichier README.md pour replacer dans l'ordre les sections de la documentation. Puis commiter les modifications.

Executer

cd ~/cloned_hello
cat README.md
git add README.md
git commit -m "Fix ordre des sections"
git status

Sortie standard

$ cd ~/cloned_hello

$ cat README.md
# Doc du projet
## Script find-dev.sh
Synopsys : find-dev <devname>
Recherche <devname> dans le carnet d'adresses des développeurs.

$ git add README.md

$ git commit -m "Fix ordre des sections"
[master 5826b6c] Fix ordre des sections	
 1 file changed, 1 insertion(+), 1 deletion(-)

$ git status
On branch master
Your branch is ahead of 'origin/master' by 1 commit.
  (use "git push" to publish your local commits)

nothing to commit, working directory clean

Le nom origin est le nom donné par défaut au dépôt distant de référence.

La commande git status nous informe que notre branche locale master a un commit d'avance sur la branche distante.

Envoyer les modifications au dépôt distant

La commande git push permet d'envoyer des modifications locales vers des dépôts distants.

Nous allons envoyer nos modifications de la branche locale master vers la branche distante origin/master du dépôt distant origin.

Executer

cd ~/cloned_hello
git push origin master
git status

Sortie standard

$ cd ~/cloned_hello

$ git push origin master
Counting objects: 7, done.
Delta compression using up to 4 threads.
Compressing objects: 100% (3/3), done.
Writing objects: 100% (3/3), 308 bytes | 0 bytes/s, done.
Total 3 (delta 2), reused 0 (delta 0)
To /home/steven/hello
   7f5dcf7..5826b6c  master -> master

$ git status
On branch master
Your branch is up-to-date with 'origin/master'.

nothing to commit, working directory clean

La vérification de notre dépôt nous informe que notre branche locale est bien synchronisée avec la branche distante.

L'envoi de modifications est possibles seulement si notre branche locale est à jour. Si la branche distante avait avancée, nous aurions été forcé de récupérer les modifications et les intégrer à notre branche pour pouvoir pousser les notres.
