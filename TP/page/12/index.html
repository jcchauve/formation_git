Itération 12 : Annuler des changements locaux

Modifier un fichier

Nous allons modifier notre fichier MAINTAINERS.txt et vérifier l'état de notre dépôt.

Executer

echo "bill@microsoft.com" >> MAINTAINERS.txt
cat MAINTAINERS.txt
git status

Sortie standard

$ echo "bill@microsoft.com" >> MAINTAINERS.txt

$ cat MAINTAINERS.txt
John Doe <jd@internet.com>
Bruce Wayne <batman@wayne-corp.com
bill@microsoft.com

$ git status
On branch master
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git checkout -- <file>..." to discard changes in working directory)

        modified:   MAINTAINERS.txt

no changes added to commit (use "git add" and/or "git commit -a")

Git a bien détecté que notre fichier MAINTAINERS.txt a bien été modifié.

Annuler le changement dans le répertoire de travail

La commande git checkout permet de restaurer notre fichier à son dernier état validé.

Nous allons restaurer l'état de MAINTAINERS.txt.

Executer

git checkout MAINTAINERS.txt
cat MAINTAINERS.txt
git status

Sortie standard

$ git checkout MAINTAINERS.txt

$ cat MAINTAINERS.txt
John Doe <jd@internet.com>
Bruce Wayne <batman@wayne-corp.com

$ git status
On branch master
nothing to commit, working directory clean
