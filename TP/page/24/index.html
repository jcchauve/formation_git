Itération 24 : Les branches distantes

Le dépôt distant origin

La commande git remote permet de gérer les dépôts distants.

Nous aller commencer par lister nos dépôts distants et afficher des information sur le dépôt origin.

Executer

cd ~/cloned_hello
git remote -v
git remote show origin

Sortie standard

$ cd ~/cloned_hello

$ git remote
origin

$ git remote show origin
* remote origin
  Fetch URL: /c/Users/Moi/hello
  Push  URL: /c/Users/Moi/hello
  HEAD branch: master
  Remote branches:
    master tracked
  Local branch configured for 'git pull':
    master merges with remote master
  Local ref configured for 'git push':
    master pushes to master (up to date)

Le nom origin est le nom donné par défaut au dépôt distant de référence.

Les branches distantes

Les branches distantes désignent les branches d'un dépôt distant. Par défaut la commande git branch ne liste pas les branches distantes. Il faut utiliser l'option -r pour les lister ou l'option -a pour lister toutes les branches.

Nous souhaitons lister toutes les branches de notre dépôt clone.

Executer

cd ~/cloned_hello
git branch
git branch -a

Sortie standard

$ cd ~/cloned_hello

$ git branch
* master

$ git branch -a
* master
  remotes/origin/HEAD -> origin/master
  remotes/origin/master

Git fourni des commandes pour pouvoir récupérer des modifications sur les branches distantes et pour envoyer des modifications réaliser sur les branches locales. Ce sont les opérations de fetch et de pull.

Les branches distantes ne sont jamais modifiées manuellement. Elles le sont seulement lorsqu'elles sont synchronisées avec le dépôt distant associé.

Une branche de suivi est une branche locale dont l'avancement est synchronisée avec une branche distante.
