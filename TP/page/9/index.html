Itération 9 : Créer des alias

Alias communs

Il est possible de définir des alias aux commandes Git existantes. Pour cela il faut modifier le fichier de configuration niveau utilisateur. Ce fichier se trouve dans votre répertoire utilisateur sous le nom .gitconfig.

Pour créer des alias il suffit de les déclarer sous un bloc alias dans le fichier de configuration de l'utilisateur.

Voici quelques alias couramment utilisés.

[alias]
  co = checkout
  ci = commit
  st = status
  br = branch

Créer un alias pour l'historique

Nous allons reprendre notre commande d'affichage personnalisé de l'historique et l'associer à l'alias hist.

[alias]
  co = checkout
  ci = commit
  st = status
  br = branch
  hist = log --pretty=format:\"%h %ad | %s%d [%an]\" --graph --date=short

Executer

git hist

Sortie standard

$ git hist
* 8b88904 2015-03-21 | Création carnet d'adresses développeurs (HEAD, master) [Steven Enten]
* 5a67ff5 2015-03-21 | Ajout fonctionnalité personnification [Steven Enten]
* 421a6a8 2015-03-21 | Initialisation du projet [Steven Enten]
