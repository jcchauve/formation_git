Itération 26 : Récupérer des modifications

Modifier le dépôt de référence

Pour illustrer la récupération de modifications, nous allons modifier notre dépôt de référence. Normalement cette modification aurait pu être envoyée par un autre développeur depuis son propre clone.

Executer

cd ~/hello
git checkout master -- .
echo "## Une section" >> README.md
git add README.md
git commit -m "Ajout nouvelle section"

Sortie standard

$ cd ~/hello

$ git checkout master -- .

$ echo "## Une section" >> README.md

$ git add README.md

$ git commit -m "Ajout nouvelle section"
[master 8f559aa] Ajout nouvelle section
 1 file changed, 1 insertion(+)

Récupérer les modifications

La branche master du dépôt de référence à avancée d'un commit. Nous allons utiliser la commande git fetch pour récupérer les modifications de la branche distante origin/master.

Nous allons récupérer les modifications du dépôt distant origin.

Executer

cd ~/cloned_hello
git fetch origin
gitk --all

Sortie standard

$ cd ~/cloned_hello

$ git hist -n 1
* 5826b6c 2015-03-22 | Fix ordre des sections (HEAD, origin/master, origin/HEAD, master) [Steven Enten]

$ git fetch origin
remote: Counting objects: 7, done.
remote: Compressing objects: 100% (3/3), done.
remote: Total 3 (delta 2), reused 0 (delta 0)
Unpacking objects: 100% (3/3), done.
Depuis /home/steven/hello
   5826b6c..8f559aa  master     -> origin/master

$ gitk --all

Fusionner les modifications

Nous constatons que la branche master distante est en avance sur la branche locale. Il faut fusionner les branches pour intégrer les modifications distantes au dépôt local.

Executer

cd ~/cloned_hello
git branch -va
git merge origin/master
git branch -va

Sortie standard

$ cd ~/cloned_hello

$ git branch -va
* master                5826b6c [en retard de 1] Fix ordre des sections
  remotes/origin/HEAD   -> origin/master
  remotes/origin/master 8f559aa Ajout nouvelle section

$ git merge origin/master
Updating 5826b6c..8f559aa
Fast-forward
 README.md | 1 +
 1 file changed, 1 insertion(+)

$ git branch -va
* master                8f559aa Ajout nouvelle section
  remotes/origin/HEAD   -> origin/master
  remotes/origin/master 8f559aa Ajout nouvelle section

$ gitk
