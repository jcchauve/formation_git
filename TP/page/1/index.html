Itération 1 : Installation

Linux

Sur une distribution Linux majeure (comme Debian) ou grand-public (comme Ubuntu), l'installation de Git est réalisable en une ligne de commande depuis le terminal (ou shell). Cette ligne lance l'installation du paquet git via le gestionnaire de paquets de la distribution utilisée (sauf FreeBSD où Git est compilé à partir de son code source).

  • Ouvrir le terminal/shell
  • Saisir la commande appropriée à votre distribution
    • Debian/Ubuntu : apt-get install git
    • Fedora : yum install git
    • Gentoo : emerge --ask --verbose dev-vcs/git
    • Arch Linux : pacman -S git
    • openSUSE : zypper install git
    • FreeBSD : cd /usr/ports/devel/git; make install
    • Solaris 11 Express : pkg install developer/versioning/git
    • OpenBSD : pkg_add git

Executer la commande git --version pour vérifier que Git a correctement été installé. Cette commande affiche la version installée de Git.

Windows

Sur Windows la procédure d'installation de Git est différente. Git est une interface en ligne de commande (Command Line Interface ou CLI). Un programme CLI s'utilise à travers un terminal. Git est conçu pour être utilisé dans un terminal Unix. Il ne peut pas être utilisé dans le "terminal" (console ou shell) Windows. Pour pouvoir utiliser Git, il faut installer des composants logiciels pour disposer d'un émulateur de terminal Unix dans Windows.

Le programme msysGit propose d'installer tout le nécessaire pour pouvoir utiliser Git sur Windows au travers d'un émulateur de terminal Unix et même depuis la console Windows (cette configuration est déconseillée pour les utilisateurs non-expérimentés).

  • Télécharger le programme d'installation sur le site officiel de msysGit
  • Lancer le programme d'installation
    • Ecran 1 : fermer tous les autres programmes ouverts et cliquer sur Next >
    • Ecran 2 : prendre connaissance de la licence logiciel (GNU GPL v2) et cliquer sur Next >
    • Ecran 3 : choisir le répertoire dans lequel installer Git et cliquer sur Next >
    • Ecran 4 : sélectionner les composants facultatifs à installer et cliquer sur Next >
    • Ecran 5 : choisir s'il faut créer des raccourcis dans le menu Démarrer et cliquer sur Next >
    • Ecran 6 : conserver l'option "Use Git from Git Bash only" sélectionnée et cliquer sur Next>
    • Ecran 7 : conserver l'option "Checkout Windows-style, commit Unix-style line endings" sélectionnée et cliquer sur Next>
    • Ecran 8 : patienter pendant l'installation de Git
    • Ecran 9 : cliquer sur Finish pour terminer le programme d'installation
  • Vérifier que Git s'est correctement installée
    • Ouvrir le programme Git Bash (raccourci dans le menu Démarrer)
    • Executer la commande git --version
    • La version installée de Git doit s'imprimer à l'écran

Le programme msysgit installe plusieurs exécutables dans le répertoire d'installation (définie dans l'écran 3). Deux exécutables doivent attirer notre attention.

  • {GIT_FOLDER}\bin\sh.exe : ouvre la fenêtre du terminal Unix (point d'entré de l'émulateur)
  • {GIT_FOLDER}\libexec\git-core\git-gui.exe : client graphique Git-Gui

Pour démarrer Git Bash – l'émulateur de shell Unix – il faut utiliser l'exécutable sh.exe.

"C:\Program Files (x86)\Git\bin\sh.exe" --login -i

Pour démarrer Git-Gui il faut utiliser l'exécutable wish.exe et passer le programme git-gui en premier paramètre.

"C:\Program Files (x86)\Git\bin\wish.exe" "C:\Program Files (x86)\Git\libexec\git-core\git-gui"

Les raccourcis pour ouvrir les programmes Git Bash et Git-Gui exécutent les commandes ci-dessus pour démarrer. Nous pouvons vérifier cette information en consultant les propriétés des raccourcis respectifs (information "Cible").

Les shells Unix et Windows

Le terminal ou shell est un programme qui à la première utilisation donne l'impression d'être perdu dans les coins les plus sombres d'un ordinateur. Une personne non-familiarisée à l'utilisation d'un shell préfère utiliser un explorateur graphique pour naviguer. Pourtant le shell offre une navigation tout aussi efficace et bien plus encore. L'explorateur graphique s'appuie d'ailleurs sur des commandes du shell.

Le tableau ci-dessous liste quelques commandes essentielles à connaître pour naviguer dans un shell Unix ou Windows.

Source : lemoda.net

Un path ou "chemin" est une chaîne de caractère désignant un répertoire ou un fichier de l'ordinateur. Un chemin Unix est différent d'un chemin Windows :

  • Windows utilise le caractère \ pour séparer les répertoires dans un chemin tandis que le caractère / est utilisé sous Unix
  • Un chemin Windows commence généralement par C:\ alors qu'un chemin Unix commence par /
  • Sous Windows et sous Unix le caractère . en début de chemin et les caractères .. dans un chemin ont la même signification
    • L'utilisation d'un . en début de chemin spécifie un chemin relatif (contrairement à l'utilisation de C:\ ou / qui spécifient un chemin absolu)
    • L'utilisation des caractères .. dans un chemin désigne le répertoire parent

Quelques exemples où nous nous trouvons dans le répertoire C:\Users\Me pour Windows et dans le répertoire /home/me/ sous Unix (ou /c/Users/Me sous l'émulateur Git Bash).

  • Lister le répertoire parent : dir .. sous Windows et ls .. sous Unix
  • Afficher le contenu de monfichier.txt dans le répertoire courant : type .\monfichier.txt sous Windows et cat ./monfichier.txt

Les chemins relatifs sont un bon moyen d'économiser des frappes supplémentaires sur le clavier.
