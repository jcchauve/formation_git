Itération 8 : Consulter l'historique

Consulter l'intégralité de l'historique

La commande git log permet de consulter l'historique du dépôt courant.

Exécuter

git log

Sortie standard

$ git log
commit 8b8890419d353fd8d53d012f3c57842f3ac1192d
Author: Steven Enten <steven@enten.fr>
Date:   Sat Mar 21 17:29:58 2015 +0100

    Création carnet d'adresses développeurs

commit 5a67ff57942cc177b3bfacc076e872a8d8d377a3
Author: Steven Enten <steven@enten.fr>
Date:   Sat Mar 21 17:28:29 2015 +0100

    Ajout fonctionnalité personnification

commit 421a6a8fdcb05445150ceffe92a13d437776cf1f
Author: Steven Enten <steven@enten.fr>
Date:   Sat Mar 21 15:15:02 2015 +0100

    Initialisation du projet

L'historique affiche la liste de tous les commits réalisés (affichés du plus récent au plus ancien).

Contrôler l'historique affiché

La commande git log supporte un certain nombre d'options pour ajuster le contenu de l'historique à afficher.

Formatter l'historique

L'option --pretty permet de formater l'affichage de chaque commit de l'historique. Plusieurs formats sont disponibles (voir les formats).

Exécuter

git log --pretty=oneline

Sortie standard

$ git log --pretty=oneline
8b8890419d353fd8d53d012f3c57842f3ac1192d Création carnet d'adresses développeurs
5a67ff57942cc177b3bfacc076e872a8d8d377a3 Ajout fonctionnalité personnification
421a6a8fdcb05445150ceffe92a13d437776cf1f Initialisation du projet

Personnaliser le format de l'historique

Exécuter

git log --pretty=format:"%h %ad | %s%d [%an]" --graph --date=short

Sortie standard

$ git log --pretty=format:"%h %ad | %s%d [%an]" --graph --date=short
* 8b88904 2015-03-21 | Création carnet d'adresses développeurs (HEAD, master) [Steven Enten]
* 5a67ff5 2015-03-21 | Ajout fonctionnalité personnification [Steven Enten]
* 421a6a8 2015-03-21 | Initialisation du projet [Steven Enten]

Filtrer l'historique

Voici plusieurs exemples de commandes permettant de filtrer l'historique affiché.

git log --pretty=oneline --max-count=2
git log --pretty=oneline --since='5 minutes ago'
git log --pretty=oneline --until='5 minutes ago'
git log --pretty=oneline --author=<your name>
git log --pretty=oneline --all

Client graphique

Les premières minutes avec Git génèrent le même sentiment que lors de la découverte du shell : le sentiment de ne pas savoir "où on en est".

Git nous permet de maintenir un historique de modifications d'un dépôt (le contenu d'un répertoire). Il fournit également différentes commandes pour intéragir avec cet historique dont la commande git log (affichant tous les commits réalisés).

L'historique peut également être consulté à travers l'utilisation de clients graphiques tiers. Gitk est l'un de ces clients de visualisation de dépôt. Il fait parti du package git-gui pour Linux et est livré avec msysGit pour Windows.

La commande gitk dans un dépôt Git ouvre une fenêtre pour visualiser l'état du dépôt.
