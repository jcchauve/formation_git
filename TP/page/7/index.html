Itération 7 : Valider des modifications

Valider les modifications

Maintenant que nous avons indexé la modification apportée à notre script hello.html, nous allons la valider dans l'historique du dépôt.

Pour cela nous allons utiliser la commande git commit.

Exécuter

git commit -m "Ajout fonctionnalité personnification"

Sortie standard

$ git commit -m "Ajout fonctionnalité personnification"
[master 5a67ff5] Ajout fonctionnalité personnification
 1 file changed, 3 insertions(+), 1 deletion(-)

Le résumé du commit (dernière ligne) nous indique le nombre de fichiers concerné par les modifications enregistrées et le nombre de lignes concernées (ajouts et suppressions).

Vérifier le statut du dépôt

Vérifions maintenant le statut de notre dépôt.

Exécuter

git status

Sortie standard

$ git status
On branch master
Untracked files:
  (use "git add <file>..." to include in what will be committed)

        MAINTAINERS.txt

nothing added to commit but untracked files present (use "git add" to track)

Git nous informe qu'il n'y a pas de modification à valider. Il nous informe également de la présence d'un fichier non-suivi. Nous allons créer un commit relatif à la création du fichier MAINTAINERS.txt.

Exécuter

git add MAINTAINERS.txt
git commit -m "Création carnet d'adresses développeurs"
git status

Sortie standard

$ git add MAINTAINERS.txt

$ git commit -m "Création carnet d'adresses développeurs"
[master 8b88904] Création carnet d'adresses développeurs
 1 file changed, 2 insertions(+)
 create mode 100644 MAINTAINERS.txt

$ git status
On branch master
nothing to commit, working directory clean

L'état de nos fichiers hello.sh et MAINTAINERS sont maintenant historisés dans la base du dépôt.

Leur état aurait pu être historisés dans le même commit. Mais sous Git il faut réfléchir sur la nature des modifications à historiser et non en terme de fichier.

Il n'est pas rare qu'un fichier modifié fasse l'objet de plusieurs commits successifs : chacun des commits apportant une modification de nature différente.

Exemple : la création d'une page HTML pourrait faire l'objet de plusieurs commits. Un premier pour l'écriture du contenu, un second pour l'écriture de la structure HTML et un troisième pour la mise en page CSS.
