Itération 10 : Récupérer d'anciennes versions

Obtenir l'empreinte des précédentes versions

Dans l'historique, chaque commit est référencé par les 7 premiers caractères de son empreinte. Ces quelques caractères suffisent à identifier un commit de manière unique.

Executer

git hist

Sortie standard

$ git hist
* 8b88904 2015-03-21 | Création carnet d'adresses développeurs (HEAD, master) [Steven Enten]
* 5a67ff5 2015-03-21 | Ajout fonctionnalité personnification [Steven Enten]
* 421a6a8 2015-03-21 | Initialisation du projet [Steven Enten]

Restaurer l'état d'un fichier

Le premier commit est identité par l'empreinte 421a6a8.

En utilisant la commande git checkout, nous allons pouvoir restaurer notre fichier hello.sh à son état initial.

Executer

cat hello.sh
git checkout 421a6a8 -- hello.sh
cat hello.sh

Sortie standard

$ cat hello.sh
#!/bin/bash
WORD="World"
[ 0 -lt $# ] && WORD="$@"
echo "Hello, $WORD"
exit 0

$ git checkout 421a6a8 -- hello.sh

$ cat hello.sh
#!/bin/bash
echo "Hello, World"
exit 0

Restaurer les fichiers à leur dernier état validé

Remplacer hello.sh par un point (.) lors de l'appel à git checkoutrestaure tous les fichiers à l'état du commit.

  • Restaurer l'état des fichiers d'un commit : git checkout 421a6a8 -- .
  • Restaurer l'état des fichiers au dernier commit : git checkout HEAD -- .

Executer

git checkout HEAD -- .
cat hello.sh

Sortie standard

$ git checkout HEAD -- .

$ cat hello.sh
#!/bin/bash
WORD="World"
[ 0 -lt $# ] && WORD="$@"
echo "Hello, $WORD"
exit 0
