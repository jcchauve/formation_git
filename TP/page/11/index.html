Itération 11 : Tagger des versions

Créer un premier tag

Un tag est un marque page vers un commit donné. C'est un moyen facile de référencer l'état du dépôt à un moment dans le temps.

Nous allons définir la version courante du dépôt comme étant la version 1 de notre projet.

Executer

git tag v1
git tag

Sortie standard

$ git tag v1

$ git tag
v1

Utiliser un tag comme référence

Le commit correspondant à la version courante du dépôt (le plus récent dans l'historique) peut maintenant être référencé par son empreinte 8b88904 et par le tag v1.

Executer

git hist
git show v1

Sortie standard

$ git hist
* 8b88904 2015-03-21 | Création carnet d'adresses développeurs (HEAD, master) [Steven Enten]
* 5a67ff5 2015-03-21 | Ajout fonctionnalité personnification [Steven Enten]
* 421a6a8 2015-03-21 | Initialisation du projet [Steven Enten]

$ git show v1
commit 8b8890419d353fd8d53d012f3c57842f3ac1192d
Author: Steven Enten <steven@enten.fr>
Date:   Sat Mar 21 17:29:58 2015 +0100

    Création carnet d'adresses développeurs

diff --git a/MAINTAINERS.txt b/MAINTAINERS.txt
new file mode 100644
index 0000000..5be7c61
--- /dev/null
+++ b/MAINTAINERS.txt
@@ -0,0 +1,2 @@
+John Doe <jd@internet.com>
+Bruce Wayne <batman@wayne-corp.com

Nous allons maintenant tagger le premier commit 421a6a8 comme étant la version v0.

Executer

git tag v0 421a6a8
git tag
git show v0

Sortie standard

$ git tag v0 421a6a8

$ git tag
v0
v1

$ git show v0
commit 421a6a8fdcb05445150ceffe92a13d437776cf1f
Author: Steven Enten <steven@enten.fr>
Date:   Sat Mar 21 15:15:02 2015 +0100

    Initialisation du projet

diff --git a/hello.sh b/hello.sh
new file mode 100755
index 0000000..94d1f4b
--- /dev/null
+++ b/hello.sh
@@ -0,0 +1,3 @@
+#!/bin/bash
+echo "Hello, World"
+exit 0

Il est maintenant possible de restaurer des fichiers à un état dans le temps référencé par un nom (plutôt qu'une empreinte).

git checkout v0 -- hello.sh
git checkout v1 -- hello.sh
